# 个人投资组合系统正式可用路线图

## 当前判断

项目已经完成了前后端主体结构、主要页面和基础统计逻辑，属于可以演示、可以本地使用的 MVP。

距离“正式可用”还差的核心不是页面数量，而是业务一致性和工程可靠性，重点是：

1. 交易没有稳定驱动账户现金与持仓变化
2. 人工编辑入口过多，容易把资产数据改乱
3. 缺少自动化验证，迭代后容易回归
4. 行情、认证、部署和监控还没有进入生产级实现

## 分阶段路线

### 第一阶段：账务闭环

目标：让系统具备稳定的记账规则，用户可以可信地维护自己的组合。

事项：

1. 交易入账后直接更新账户现金和持仓数量
2. 买入校验账户可用现金，卖出校验持仓数量
3. 将交易改为不可直接编辑 / 删除，改用反向交易修正
4. 在前端明确提示“交易驱动持仓”，降低误操作
5. 为核心结算逻辑补自动化测试

状态：本次执行

### 第二阶段：持仓维护分层

目标：把“行情更新”和“仓位变更”彻底拆开，减少手工改数据的风险。

事项：

1. 持仓页面只允许更新展示字段和现价
2. 初始持仓导入改成独立流程
3. 增加批量导入持仓 / 交易能力
4. 为调整行为增加操作说明和审计字段

状态：进行中（已完成事项 1、2、3）

### 第三阶段：真实行情与估值

目标：让收益和资产分布不再依赖手工现价维护。

事项：

1. 接入真实行情源
2. 定时刷新持仓现价
3. 引入缓存层（Redis）
4. 将快照生成改为定时任务而不是只在请求时更新

### 第四阶段：可靠性与安全

目标：从个人脚本型项目提升到长期可维护系统。

事项：

1. 登录认证和基础权限
2. 数据迁移和环境配置规范化
3. API / 结算 / 前端关键流程测试体系
4. 错误监控、审计日志、备份策略

### 第五阶段：增强能力

目标：提升长期使用体验。

事项：

1. 报表导出与绩效归因
2. 更完整的 AI 分析上下文
3. 多账户、多策略视角
4. 标签、策略归类、复盘记录

## 本次执行范围

本次只落地第一阶段中的最高优先级：

1. 交易记账结算服务
2. 买入 / 卖出校验
3. 交易不可编辑 / 删除的规则收口
4. 前端交易页交互调整
5. 自动化测试
